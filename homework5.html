Introduction

The first part reviews measures of location and when each one is useful. The second part derives simple recurrence formulas to update mean and variance when a new value arrives. An optional note compares online updates with the classical full recomputation in terms of memory and time.

Part 1: Measures of Location

Arithmetic mean

Definition: average of all values. Formula for values x1,…,xn: mean = (x1 + x2 + … + xn) / n. Use when data are roughly symmetric with no extreme outliers. Example: average temperature over a week.

Median

Middle value after sorting. Robust to outliers and skewed data. Use for incomes, house prices, waiting times. Example: for 2, 3, 9, 12, 100 the median is 9 while the mean is influenced by 100.

Mode

Most frequent value. Works for discrete or categorical data. Use for the most common customer choice, most frequent error code, most common eye color.

Weighted mean

Some observations count more than others. Formula with weights wi: weighted mean = (Σ w_i x_i) / (Σ w_i). Use for grade point averages or when combining estimates with different reliability.

Geometric mean

Multiplicative average: GM = (x1 · x2 · … · xn)^(1/n) for positive values. Use for growth rates and returns over time since compounding is multiplicative.

Harmonic mean

Reciprocal of the average of reciprocals: HM = n / (Σ 1/x_i) for positive values. Use for averages of rates such as speed or throughput.

Trimmed mean

Average after removing a small fraction of the smallest and largest values. Use when you want a mean that is more robust than the arithmetic mean yet keeps most of the data.

Part 2: Online mean and variance

Online mean

Let mn be the mean after n values. When a new value xn arrives: m_n = m_{n-1} + (x_n − m_{n-1}) / n. This needs only the previous mean and the count.

Online variance with Welford update

Maintain three values: count n, mean m, and M2, where M2 is the running sum of squared deviations. For each new x: 

// start
n = 0
m = 0
M2 = 0

// update for each new x
n = n + 1
delta = x - m
m = m + delta / n
delta2 = x - m
M2 = M2 + delta * delta2

// results at any time
sample_variance   = M2 / (n - 1)   // for n ≥ 2
population_variance = M2 / n       // if population

This method is stable in floating point arithmetic and requires constant memory since it does not store the whole dataset.

Derivation sketch

Write the new mean as the previous mean plus the correction created by the new value. The M2 update follows from expanding the new sum of squared deviations and simplifying so that only the old mean, the new mean, and the new value appear.

Optional: online vs classical approach

Classical computation reads all values, then computes mean and variance from the full sums. This uses more memory and is slower for streams. Online updates process one value at a time, keep only n, m, M2, and are well suited for real time systems, embedded devices, and continuous logging. A single pass is usually enough which reduces time and avoids storing the array.

Conclusion

Measures of location summarize where data are centered and each choice fits a different situation. Online updates provide a practical way to keep mean and variance current while data arrive, with low memory and good numerical behavior.
