1) Introduction

This homework extends the idea of text analysis and substitution ciphers by introducing the RSA encryption scheme. We build on the earlier Caesar-cipher exercise and study how letter frequencies behave under a true modular-arithmetic cipher. The work combines basic number theory, probability, and linguistic statistics.

2) RSA algorithm overview

RSA is based on modular exponentiation using a public and private key pair. For demonstration we use tiny primes, following the classroom example.

StepFormula / Value
Choose primesp=29, q=31
Compute modulusN=p·q = 899
Euler totientφ(N)=(p−1)(q−1)=840
Choose public exponente=17 (coprime with 840)
Find private exponentd=593 since 17·593 ≡ 1 (mod 840)

Encryption and decryption: C = Me mod N, M = Cd mod N. Euler’s theorem ensures correctness because Mφ(N) ≡ 1 (mod N) for coprime M,N.

Tip. Small numbers are used here only for illustration. Real RSA implementations employ primes hundreds of digits long.

Unlike Caesar cipher, RSA produces outputs that are uniformly distributed modulo N and destroy frequency patterns. Therefore, frequency analysis fails—demonstrating the strength of modular arithmetic encryption.

4) Worked toy example

Take M = 65 (letter ‘A’). Encrypt: C = 6517 mod 899 = 587. Decrypt: M = 587593 mod 899 = 65 → ‘A’. The round-trip confirms the algorithm.

5) Algorithm summary (pseudocode)

Input: message M (integers), keys (e, d, N)
for each block x in M:
    encrypt: c = x^e mod N
    decrypt: m = c^d mod N
output ciphertext C and recovered message

6) Interpretation

The exercise highlights the difference between statistical ciphers and number-theoretic encryption. While Caesar can be reversed by analyzing letter distribution, RSA cannot because ciphertext values behave like random numbers. Still, probability remains relevant: its security relies on the difficulty of factoring N into p and q.
