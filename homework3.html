Introduction

This homework connects cryptography with statistical analysis of language. We move from the distribution-based reasoning used with Caesar cipher to the number-theoretic mechanism of RSA. A short Greek text from C. P. Cavafis is used to illustrate encoding, encryption, decryption, and how frequency patterns behave.

RSA Overview (Toy Numbers for Didactics)

Small primes are used purely for demonstration; real RSA uses very large primes.

StepValue / Formula
Choose primesp = 29, q = 31
ModulusN = p·q = 899
Euler totientφ(N) = (p−1)(q−1) = 28·30 = 840
Public exponente = 17 (gcd(17, 840) = 1)
Private exponentd = 593 since 17·593 ≡ 1 (mod 840)

Encryption: C = Me mod N • Decryption: M = Cd mod N • By Euler’s theorem, this round-trip recovers M.

Worked Example

Plain letter: ‘Α’ → M = 1 (using Α→1 mapping)

Encrypt: C = 117 mod 899 = 1

Decrypt: M = 1593 mod 899 = 1 → ‘Α’

With real text, we process each normalized Greek letter to a number, encrypt, and later decrypt back to letters.

Plain vs Cipher: Frequency Discussion

In Caesar cipher, plaintext letter frequencies leak into ciphertext, enabling frequency attack. In RSA, ciphertext values are residues modulo N from exponentiation; they do not preserve the plaintext’s letter frequency pattern. Hence, frequency analysis that works for Caesar does not apply to RSA.

Conclusion

    

      The exercise shows a clear contrast: descriptive statistics enable attacks on simple substitution ciphers,
      but number-theoretic encryption like RSA resists such methods. Language distributions remain valuable for
      classical cryptanalysis, while modern cryptography relies on hard arithmetic problems.
