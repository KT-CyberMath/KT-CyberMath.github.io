Homework 10 — From Bernoulli Trials to the Poisson Process

1) Discrete Approximation of a Counting Process

We consider a time interval [0, T] and a process that counts the number of events or successes that occur in this interval. Each event is assumed to happen independently of the others and the average rate of events is λ per unit time. For concreteness we can take T equal to 1.

To simulate the process we approximate continuous time with a fine discrete grid. We divide [0, T] into n sub intervals of equal length Δt = T / n. In each sub interval we perform a Bernoulli trial with probability of success equal to λ Δt. If a success occurs we record an event in that sub interval, otherwise no event occurs there.

For large n this simulation gives a discrete time process that approximates a continuous time counting process with constant rate λ.

2) Distribution of the Number of Events

Let Nn(t) be the number of events observed in the simulation up to time t in [0, T]. For a fixed t, approximately k = n t / T sub intervals fall inside [0, t]. Each of these k sub intervals contains an event with probability λ Δt, independently of the others. Therefore the number of events in [0, t] has a binomial distribution:

Nn(t) ∼ Binomial(k, λ Δt) with k ≈ (n t) / T and Δt = T / n

If we keep t and λ fixed and let n grow to infinity, then k grows and the success probability in each trial becomes λ / n. In this limit we obtain the classical Poisson approximation of the binomial distribution. The law of Nn(t) converges to a Poisson distribution with parameter λ t:

Nn(t) ⇒ N(t) , with N(t) ∼ Poisson(λ t)

Thus the simulated counting process converges in distribution to a Poisson counting process.

3) Identification as a Poisson Process

The limiting process N(t), for t in [0, T], has the following properties:

  • N(0) = 0.
  • N(t) increases by jumps of size one at random times, each jump representing a new event.
  • Independent increments: for disjoint time intervals the numbers of events are independent random variables.
  • Stationary increments: for s less than t the increment N(t) minus N(s) depends only on t minus s and follows a Poisson(λ (t − s)) law.

A counting process with these properties is called a Poisson process with rate λ. The discrete simulation based on Bernoulli trials over small sub intervals is a standard way to construct it by approximation.

4) Theoretical Properties

For every fixed t the random variable N(t) follows a Poisson distribution with parameter λ t. Its probability mass function is

P(N(t) = k) = e−λ t (λ t)k / k! , k = 0, 1, 2, ...

The mean and variance of N(t) are both equal to λ t:

E[N(t)] = λ t, Var(N(t)) = λ t

If we look at an increment over an interval of length h, then N(t + h) minus N(t) follows a Poisson(λ h) law and is independent of the past. These properties match the behavior that appears in the simulation when n is large.

5) Inter Arrival Times and Exponential Law

Another way to describe the same process is to look at the waiting time between two consecutive events. Let T1 be the time until the first event occurs. In a Poisson process with rate λ this waiting time has an exponential distribution with parameter λ:

P(T1 > t) = e−λ t , t ≥ 0

The same holds for the inter arrival times between any two events. The number of events up to time t is then the largest k such that the sum of the first k waiting times is less than or equal to t. This description highlights the relation between the Poisson process and the exponential distribution.

6) Interpretation of the Rate λ

The parameter λ is the intensity of the process. It has several equivalent meanings:

  • It is the expected number of events per unit time. On average we see λ events in a time interval of length one and λ t events in a time interval of length t.
  • Its inverse 1 / λ is the expected waiting time between two consecutive events.
  • It controls the spread of the process. Since Var(N(t)) = λ t, larger λ produces more variability but in relative terms the fluctuations around the mean become smaller when t grows.

In applications λ is interpreted as arrival rate, failure rate or intensity of requests, depending on the context. The Bernoulli simulation over small time steps provides a concrete way to see how this rate parameter governs both the distribution of counts and the timing of events.
